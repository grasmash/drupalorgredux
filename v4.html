Why AI agents prefer Drupal

Structured content is
AI's best friend

AI agents don't just generate content in Drupal. They understand its structure, respect its permissions, and operate within its workflows.

🤝

MCP Protocol

Native Model Context Protocol support. Claude, Cursor, Copilot — any MCP-compatible agent connects and operates your site.

🎯

Context Control Center

Define your brand voice, target audiences, and key messages in one place. Every AI agent draws from this single source of truth.

📋

Human-in-the-Loop

Every AI action is queued for review. Batch approvals, audit trails, revision history. Nothing reaches production without human consent.

🧩

Structured & API-First

Typed content models, JSON:API in core, Drush for CLI automation, and semantic config that AI can read, reason about, and modify. No scraping. No guessing.

🔒

Granular Permissions

Scope agent access precisely: "create articles, edit own drafts, can't touch users." The most granular permission system of any major CMS.

🌐

Any AI Provider, No Lock-in

OpenAI, Anthropic, Google Gemini, Azure, AWS Bedrock, self-hosted models. Swap providers without code changes. Self-host if you need to.

Developer escape hatch

Are you a developer?

Skip the GUI. Two commands and you're running.

curl -fsSL https://ddev.com/install.sh | bash
mkdir my-site && cd my-site && ddev config --project-type=drupal \
  --docroot=web && ddev composer create-project drupal/cms && ddev launch
